35.1 CCA mapping — findings and fix

MapQuery produced two embedding outputs: internal reference-PCA coordinates used for label transfer scores (saved to disk), and UMAP projection coordinates (ref.umap) used for visualization. The internal projection coordinates were the relevant input for downstream aggregation.

A critical issue was identified in the MapQuery output: each projected dimension was L2-normalized to unit column norm, which removed PCA eigenvalue decay. All 15 CCA dimensions therefore showed near-identical standard deviation (~0.00545), which would force SEACells to weight each dimension equally regardless of biological signal strength.

We tested multiple reference PCA parameter settings (DIMS: 1:15, 1:20, 1:25, 1:30; k.anchor: 5, 10, 15, 20; k.filter: 100, 150, 200; k.score: 30, 50). The F3000_D1-15_cca setup with default k-parameters remained the best-performing configuration.

35.2 Thoughts: SEACells aggregation — batch effect decision

SEACells was initially run on each batch separately, and the aggregation results were poor. A pooled run across all batches is currently in progress to assess whether shared biological structure is recovered more effectively.
